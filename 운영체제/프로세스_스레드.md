
## 1. 프로세스
### 정의
* 프로세스는 프로그램이 메모리에 적재되어있는 상태로 운영체제로부터 CPU 자원을 할당받아
작업하는 단위이다.

### 메모리
* stack , heap , data , code 영역의 메모리영역을 가지고 프로세스간 메모리를 공유하지 않는다.  
* 프로세스간 메모리를 공유하지 않기때문에 context switching과 통신 비용이 발생한다.

### IPC(Inter Process Communication)
* message passing 방식으로 커널을 통해 데이터를 전송한다. 커널을 통해 데이터를 전송하기때문에
성능면에서 좋지 않으나 동기화문제가 발생하지 않는다.
* shared memory 방식으로 시스템 호출을 통해 데이터 영역에 공유메모리 영역을 할당한다.
성능면에서 좋으나 동기화문제가 발생한다.

### CPU bound와 IO bound
* CPU burst는 프로세스가 CPU를 한번에서 연속적으로 연산을 실행하는 시간을 의미하고 
IO burst는 프로세스가 IO 작업을 요청하고 결과를 기다리는 시간을 의미한다.
* CPU bound 프로세스에서 적정 스레드 수는 CPU 코어수+1 이다.

## 2. 스레드
### 정의
* 프로세스 내에세 실행되는 실행주체로 프로세스는 1개 이상의 스레드를 가진다.

### 메모리
* heap , data , code 영역의 메모리를 공유하고 독립적인 stack 영역을 가진다.
* stack을 제외한 메모리 영역을 공유하기 때문에 context switching 비용이 적게 든다. 
하지만 메모리를 공유하기때문에 동기화문제에 유의해야한다.

## 인터뷰
### Copy On Write란?
* 리소스를 공유하는 상태에서 리소스에 대한 쓰기작업이 일어나는 순간 리소스의 복사본을 만드는 방법이다.
fork를 사용시 자식 프로세스는 부모 프로세스와 같은 페이지를 공유하게 되는데 자식 프로세스에서 페이지 수정이
일어난다면 페이지의 복사본을 만들어서 수정한다.

### 프로세스 비정상 종료
* 좀비 프로세스는 부모 프로세스가 wait을 호출하기전에 자식 프로세스가 실행을 종료하면
자식 프로세스는 좀비 프로세스가 되어서 프로세스 테이블 엔트리에 남게 된다.
* 고아 프로세스는 부모 프로세스가 먼저 종료된 자식 프로세스로 커널이 자동으로 
부모를 init 프로세스로 설정해준다.
